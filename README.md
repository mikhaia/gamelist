# GameList

Русскоязычное Laravel-приложение для личных списков игр с публичными read-only ссылками, импортом из текста и интеграцией HowLongToBeat.

## Возможности

- регистрация с необязательным уникальным email, вход по логину или email и восстановление пароля шестизначным OTP-кодом;
- публичные профили с обложкой, аватаром, любимыми играми и игровой статистикой;
- направленная дружба без подтверждения и отдельный список входящих добавлений;
- постоянные уведомления о дружбе, публичных списках, играх и прохождениях;
- несколько списков на пользователя;
- публичные ссылки формата /login/list-slug;
- карточный и компактный режимы;
- фильтрация по одному или нескольким статусам;
- копирование текущей выборки в буфер обмена в виде «- Название игры»;
- статусы прохождения и платформы Nintendo Switch, Steam, PC, PlayStation, Xbox;
- необязательные даты начала и окончания игры;
- общая игровая история в виде хронологии начала и завершения игр;
- публичные страницы каталожных игр с общими счётчиками статусов, оценками и Markdown-отзывами; ручные игры остаются без отдельной страницы;
- быстрое добавление игры с её страницы в один из собственных списков;
- импорт до 100 игр с очисткой Markdown и предпросмотром дубликатов;
- поиск обложек и времени прохождения через отдельный GameCatalog-сервис;
- локальный MySQL-кэш поиска с мгновенной первой выдачей, фоновым обновлением и общедоступной страницей `/search` без перезагрузки;
- каталог с lazy load по 20 игр и быстрым добавлением нескольких игр в выбранный список;
- ручное добавление при недоступности HowLongToBeat;
- обложки из каталога, URL или файла с локальной WebP-оптимизацией;
- фоновые обложки списков, пользовательские аватары, смена email и пароля;
- письма через Resend о новых друзьях и дружелюбные напоминания после месяца отсутствия;
- русская локализация, подготовленная к добавлению других языков;
- собственные favicon, Apple Touch Icon и web app manifest.

## Локальный запуск

Требования: PHP 8.3+, Composer, Node.js, MySQL и расширение GD.

1. Скопировать .env.example в .env и выполнить php artisan key:generate.
2. Создать MySQL-базу gamelist. Стандартные локальные параметры уже указаны в .env.example.
3. Выполнить composer install, npm install, php artisan migrate и php artisan storage:link.
4. Собрать ассеты командой npm run build или запустить npm run dev.
5. Запустить приложение командой php artisan serve.

## Хостинг без Node.js

Production-ассеты из public/build создаются локально и добавляются в Git. На сервере не нужно запускать npm install или npm run build: достаточно получить репозиторий, установить PHP-зависимости через Composer и выполнить миграции Laravel.

После изменений интерфейса перед коммитом нужно локально выполнить npm run build и добавить обновлённые файлы public/build в тот же коммит.

Для почтовых уведомлений на хостинге нужно задать `MAIL_MAILER=resend`, `RESEND_API_KEY`, `MAIL_FROM_ADDRESS=gamelist@div.kz`, корректный `APP_URL` и подтвердить домен отправителя в Resend. Ключ хранится только в окружении и не добавляется в Git.

Проверка неактивных пользователей запускается планировщиком Laravel ежедневно. На хостинге должна выполняться стандартная cron-команда `php artisan schedule:run` каждую минуту.

Подробный план находится в PLAN.md, состояние задач — в TODO.md.
