# План разработки GameList

## Цель

Создать русскоязычное Laravel-приложение для ведения нескольких игровых списков с публичными read-only ссылками, импортом из текста и локально сохранёнными обложками.

## Этапы

1. Основа проекта: Laravel 13, MySQL, Tailwind CSS 4, русская локаль.
2. Данные и доступ: пользователи с уникальным логином, списки, игры, платформы и статусы.
3. Основные сценарии: регистрация, вход, CRUD списков и игр, публичный просмотр.
4. Импорт: очистка Markdown-префиксов, предпросмотр, обнаружение повторов, подтверждение.
5. Каталог: адаптер над aneeskhan47/php-howlongtobeat-api и ручной fallback.
6. Обложки: выбор из HowLongToBeat, URL или загрузка; скачивание, WebP-оптимизация, локальное хранение.
7. Интерфейс: адаптивная тёмная glassmorphism-тема, карточный и компактный режимы, Material Icons.
8. Качество: feature-тесты, форматирование, сборка фронтенда, миграции MySQL.

## Архитектурные решения

- HowLongToBeat подключён через GameCatalog-контракт; библиотека не используется в контроллерах.
- Публичный адрес списка: /{login}/{slug}; slug уникален внутри аккаунта.
- Статусы и платформы хранятся как стабильные строковые ключи, а подписи берутся из файлов локализации.
- Все внешние изображения преобразуются в WebP и обслуживаются с локального диска public.
- Недоступность HowLongToBeat не блокирует ручное добавление игры.
